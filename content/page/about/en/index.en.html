Frequently used tools

Currently Icons are mainly made using Illustrator, which is an old drawing software of Adobe.

In the case of paper media, color correction by CMYK is also performed.

When there is no material, I am making material from 1 with almost Illustrator.

Currently it is using Photoshop, an old Adobe photo retouching tool, and now we mainly produce banner images and process / modify photos.

Because it is almost impossible to use the raw material as it is, it is mainly used for clipping of people and material creation.

Currently I mainly design the screen layout using XD which is a prototype making tool provided by Adobe.

It is very useful because it is equipped with plug-in implementation, comps and design specifications.

We rarely create videos using Adobe Animate (formerly Flash).

We do not make sophisticated animations, but make animations with simple motion and picture-story style in order to compensate for difficult parts that are difficult to transmit with only still images.

We convert the animation we make into code, extract only the necessary parts, and implement it.

Coding is mainly implemented using VIM.

In addition to basic remodeling of. vimrc, I install plug-in and rework it to my own VIM (tool uses dein)

Basically it does not use the mouse, it is all done by keyboard operation.

Terminals are mainly using Hyper Term.

In addition to being able to freely change appearance using themes, we plug in and plug it in to create our own terminal.

However, since it does not support Japanese input, we use a Mac standard terminal to put messages in Japanese at Git's Commit.

To be exact, it is a web service, but I prefer netlify, which monitors Git's repository and automatically goes to Deploy and publish it on a one-stop basis.
